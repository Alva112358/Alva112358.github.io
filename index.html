LeetCode 927 Three Equal Parts

题目描述

题目链接：https://leetcode.com/problems/three-equal-parts/

You are given an array arr which consists of only zeros and ones, divide the array into three non-empty parts such that all of these parts represent the same binary value.

If it is possible, return any [i, j] with i + 1 < j, such that:

  • arr[0], arr[1], ..., arr[i] is the first part,
  • arr[i + 1], arr[i + 2], ..., arr[j - 1] is the second part, and
  • arr[j], arr[j + 1], ..., arr[arr.length - 1] is the third part.
  • All three parts have equal binary values.

If it is not possible, return [-1, -1].

Note that the entire part is used when considering what binary value it represents. For example, [1,1,0] represents 6 in decimal, not 3. Also, leading zeros are allowed, so [0,1,1] and [1,1] represent the same value.

Constraints:

  • 3 <= arr.length <= 3 * 104
  • arr[i] is 0 or 1

题目要求找到满足限制条件的下表 $i$ $j$，使得 $[0,i-1]$，$[i, j]$，$[j+1, n-1]$ 的二进制表示的值相同。

解决方案

可以将问题转换为相同的 1 的个数：即三个部分的二进制表示应该拥有相同个数的 1，并且每一部分的首个 1 以后出现的数字顺序相同，即可验证相等，写成过程的表达如下所示：

  1. 首先需要计算数组中 1 出现的次数 $c$；
    • 如果 $c = 0$，则数组可以表示成相等的三部分，并且每部分的十进制值都是 0，此时只需要返回 $[0,\text{arr.size()}-1]$.
    • 如果 $\text{mod}(c, 3) \neq 0$，则不可能表示成拥有相同 1 的三部分，因此返回结果 $[-1,-1]$.
  2. 遍历数组，记第 1 次出现 1 的下标未 $\text{start}$，第 $c/3+1$ 次出现的 1 的下标未 $\text{mid}$，第 $2c/3+1$ 次出现的 1 的下标为 $\text{end}$.
  3. 验证 $[\text{start},\text{mid}-1]$，$[\text{mid},\text{end}-1]$，$[\text{end},\text{arr.size()-1}]$ 相同。

时间复杂度

统计 $1$ 的个数和获得 $\text{start}$，$\text{mid}$，$\text{end}$ 都需要遍历一次数组，时间复杂度为 $O(2n)$.

验证数组需要遍历 $1/3$ 的数组长度，时间复杂度为 $O(n/3)$.

因此，算法的时间复杂度为 $O(n)$.

Given an n x n matrix where each of the rows and columns are sorted in ascending order, return the kth smallest element in the matrix.

Note that it is the kth smallest element in the sorted order, not the kth distinct element.

Constraints:

  • n == matrix[i].length
  • 1 <= n <= 300
  • -109 <= matrix[i][j] <= 109
  • All the rows and columns of matrix are guaranteed to be sorted in non-decreasing order.

题目需求是求出二维数组中第 $k$ 小的数，并且二维数组满足每一行和每一列都是单调递增的。

解决方案

Heap

使用最小堆实现的优先队列 $Q$ 来解决本问题，有点像类似解决合并 $k$ 个有序链表的算法，我们的算法步骤如下所示：

LeetCode 556 Reshape the Matrix

题目描述

题目链接：https://leetcode.com/problems/reshape-the-matrix/

In MATLAB, there is a handy function called reshape which can reshape an m x n matrix into a new one with a different size r x c keeping its original data.

You are given an m x n matrix mat and two integers r and c representing the row number and column number of the wanted reshaped matrix.

The reshaped matrix should be filled with all the elements of the original matrix in the same row-traversing order as they were.

If the reshape operation with given parameters is possible and legal, output the new reshaped matrix; Otherwise, output the original matrix.

题目问如何将一个 $m \times n$ 的矩阵转换成 $r \times c$ 的矩阵。

解决方案

主要需要明白二维数组和一维数组的转换关系。

  1. 二维数组转换成一维数组

    若将 $m \times n$ 的矩阵 $A_0$ 转换成一维数组 $A_1$，则一维数组中的第 $k$ 个元素在 $m \times n$ 矩阵中为 $(k/n, k\%n)$，即 $A_1[k]=A_0[k/n][k\%n]$.

  2. 一维数组转换成二维数组

    若将长度 $N = m \times n$ 的矩阵 $A_1$ 转换成 $r \times c$ 的矩阵 $A_2$，则转换的公式为 $A_1[k] = A_2[k/c][k\%c]$.

综合上述的两条公式，我们可以得到 $A_2[k/c][k\%c] = A_0[k/n][k\%c]$.

LeetCode 1338 Reduce Array Size to The Half

题目描述

题目链接：https://leetcode.com/problems/reduce-array-size-to-the-half/

Given an array arr. You can choose a set of integers and remove all the occurrences of these integers in the array.

Return the minimum size of the set so that at least half of the integers of the array are removed.

题目给定一个数组 $A$，选择一个整数的集合 $S$，删除 $A$ 中所有 $S$ 中存在的元素，新集合 $A’$ 满足 $|A’| \leq |A|/2$，返回最小的集合 $S$.

解决方案

哈希 + 排序

采用哈希+排序的思路，首先统计数组中各元素的频率，然后对这些频率进行排序，然后依次删除频率最高的元素，直至数组的元素个数小于或者等于原来的一半。

算法过程如下：

  1. 创建哈希表，统计数组 $A$ 中各元素的频率；
  2. 将哈希表中的元素存放到新的数组中，并对数组进行排序；
  3. 若数组 $A$ 的长度为 $N$，则依次删除数组中频率最高的频率，从 $N$ 中进行扣除，直到数组中剩余的元素小于或者等于原来的一半。

算法分析

时间复杂度

假设数组 $A$ 的长度为 $n$.

  1. 统计 $A$ 中元素的频率，需要遍历一遍原来的数组，时间复杂度为 $\Theta(n)$.
  2. 存放哈希表中的频率值到新的数组中，时间复杂度为 $O(n)$.
  3. 排序的时间复杂度为 $O(n\lg n)$.
  4. 获取最终结果的时间复杂度为 $O(n)$.

综合，算法的时间复杂度为 $O(n \lg n)$.

空间复杂度

额外的空间主要用到了哈希表和存放频率的数组，空间复杂度为 $O(n)$.

Application

  • Autocomplete: 搜索时能够根据前缀进行自动填充，根据当前输入的前缀获取提示。
  • Spell Checker: 词法纠错，判断某个单词是否拼写错误，只需要 search 相关的单词是否在词库里面，即可检查是否存在拼写错误。

Trie 的实现细节与时间复杂度

本文的实现参考 LeetCode 208 题 Implement Trie. 需要实现 Trie 的插入 insert, 搜索 search, 判断前缀 starWith 三个方法，我们的实现基于字典树，即默认每个节点都有 26 个子节点，表示下一个单词可能是 26 个英文单词，同时我为每个节点设置一个标签 end_ 表示此处是否是一个单词的结尾。

时间复杂度

search: 时间复杂度为 $O(L)$，其中 $L$ 表示单词的长度，在搜索的过程中，我们需要类似于插入不断地获取子节点，但是遇到空节点时会提前返回。

starWith: 时间复杂度为 $O(L)$，道理与 search 相近。
