LeetCode 378 Kth Smallest Element in a Sorted Matrix

题目描述

题目链接：https://leetcode.com/problems/kth-smallest-element-in-a-sorted-matrix/

Given an n x n matrix where each of the rows and columns are sorted in ascending order, return the kth smallest element in the matrix.

Note that it is the kth smallest element in the sorted order, not the kth distinct element.

Example 1:

Constraints:

  • n == matrix.length
  • n == matrix[i].length
  • 1 <= n <= 300
  • -109 <= matrix[i][j] <= 109
  • All the rows and columns of matrix are guaranteed to be sorted in non-decreasing order.
  • 1 <= k <= n2

题目需求是求出二维数组中第 $k$ 小的数，并且二维数组满足每一行和每一列都是单调递增的。

解决方案

Heap

使用最小堆实现的优先队列 $Q$ 来解决本问题，有点像类似解决合并 $k$ 个有序链表的算法，我们的算法步骤如下所示：

  1. 将矩阵 $M$ 中每一行的第一个元素加入到 $Q$.
  2. 取出 $Q$ 中的顶部元素，并将相应行的下一个元素加入到 $Q$.
  3. 步骤 2 重复 $k-1$ 次。

Binary-Search

在本题中，二分查找采用范围查询，已知矩阵 $A$ 的最大值为 $A_{min}$，最大值为 $A_{max}$.

  • 使用二分查找，找出 $A^{x}$，使得 $A$ 中小于或者等于 $A^x$ 的元素的数目为 $k$.
  • 直至找到最小的 $A^x$.

下面我们证明 $A^*$ 在 $A$ 中必然存在：

  • 如果 $A’ = (A_{min}+A_{max})/2$，且 #$(A_i \leq A’) > k$，则说明仍然存在 $A^x < A’$，使得 #$(A_i \leq A^x) = k$，此时只需要缩小 $A_{max}$ 的范围即可；
  • 如果 $A’ = (A_{min}+A_{max})/2$，且 #$(A_i \leq A’) < k$，则说明仍然存在 $A^x > A’$，使得 #$(A_i \leq A^x) = k$，此时只需要增大 $A_{min}$ 的范围即可；
  • 如果 $A’ = (A_{min}+A_{max})/2$，且 #$(A_i \leq A’) = k$，此时 $[A^x,A’]$ 间的值都可以满足 #$(A_i \leq A^x) = k$，我们只需要缩小 $A_{max}$ 就可以找到 $A$ 中存在的值。

算法分析

Heap
时间复杂度
  • C++ 代码中的 13-15 行遍历 $A$ 的行，时间复杂度 $O(n)$.
  • C++ 代码中的 17-25 重复 $k-1$ 次获取最小元素的操作，每次操作均需要维护一个大小为 $n$ 的最小堆，时间复杂度为 $O(k \lg n)$.

综合总的时间复杂度为 $O(n+k \lg n)$.

考虑到 $k$ 的取值范围 $[1, n^2]$，时间复杂度也可以是 $O(n^2 \lg n)$.

若 $k < n$，则时间复杂度为 $O(n \lg n)$.

空间复杂度

使用堆的算法需要维护一个大小为 $n$ 的最小堆，空间复杂度为 $O(n)$.

Binary-Search
时间复杂度
  • 二分查找的时间复杂度为 $O(\lg (A_{max}-A_{min}))$.
  • 内部嵌套的寻找小于或者等于算法的时间复杂度为 $O(n)$.

因此，时间复杂度为 $O(n\lg (A_{max}-A_{min}))$.

空间复杂度

此算法不需要额外的空间，因此空间复杂度为 $O(1)$.
